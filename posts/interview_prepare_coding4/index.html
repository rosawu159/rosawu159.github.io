稍微在日常準備一些技術題目以備不時之需。以工程師面試來說，面試主要會分成：

  1. 演算法
  2. 概念理解
  3. 系統設計

演算法基本上是以leetcode刷題為主，而系統設計主要是看Alex Xu的文章。這裡也會時不時將軟體工程師需要具備的概念進行整理。

QUESTION: 給定以下 HTML 片段，哪些 XPath 定位器唯一標識搜尋按鈕？

OPTIONS:
1. //input
2. //input[@id=’searchButton’]
3. //div/div/input[contains@class, ‘search’)]

ANSWER:
1/3/4

QUESTION: Given the following HTML snippet, which CSS locators uniquely identify the search button?

OPTIONS:
1. input#searchButton
2. outer > #searchButton
3. •outer #searchButton
4. inner > #searchButton
5. outer * #searchButton
6. .inner :nth-child(2)

ANSWER:
1/2/3/4/5

QUESTION: There exists a below class called Users to perform CURD operation on the user details currently works well with MySQL DB.
Assume there is a new change coming up to store it using a different web service, how would you refactor the code so that you can unit test it and well accommodate any similar changes in future.

OPTIONS:
1. Replace MySQL with a new web service
2. Replace MySQL with a new web service and mock the web service to perform unit test
3. Create an interface for storage and have different implementations
4. Remove the MySQL dependency and move storage details to the Users class

ANSWER:
3
